# CONTRIBUTING

## Назначение

Этот документ описывает только процесс внесения изменений в репозиторий.
Установка и запуск проекта описаны в `README.md`.

## Базовый workflow

1. Создай рабочую ветку от `main`.
2. Внеси изменения.
3. Прогони проверки локально.
4. Сделай commit и push.
5. Открой Merge Request.

## Обязательные проверки

Перед push обязательно:

```bash
make project-check
```

Автоматически запускается:

- `pre-commit` на commit
- `pre-push` на git push запускает:
  - `make type-check`

Если хук упал, исправь проблему и повтори commit/push.

Когда в проекте появятся тесты, имеет смысл вернуть обязательный pre-push запуск тестов.

## Требования к документации и docstring

- Текст документации в репозитории веди на русском языке.
- Для каждого публичного Python-модуля обязателен module docstring.
- Для каждого публичного класса и функции обязателен docstring с кратким описанием.
- Для функций в docstring указывай секции `Args`, `Returns` и `Raises` при наличии соответствующих аргументов, возвращаемого значения и исключений.
- Проверка docstring обязательна в линтере (`ruff` правила `D` с `convention = "google"`).

## Соглашение по commit message

Рекомендуемые типы:

- `feat`: новая функциональность
- `fix`: исправление бага
- `refactor`: рефакторинг без изменения поведения
- `test`: тесты
- `docs`: документация
- `chore`: служебные изменения
- `perf`: оптимизация производительности
- `ci`: изменения CI/CD
- `revert`: откат

Примеры:

- `feat(bot): добавил новый режим поиска`
- `fix(search): исправил ранжирование результатов`
